================================================================================
                    DATA FLOW EXAMPLE
================================================================================

EXAMPLE: User Creates a New Task
---------------------------------

1. USER ACTION:
   - User fills task form on ProjectPage.jsx
   - Clicks "Create Task" button

2. FRONTEND (React):
   - handleCreateTask() function called
   - Axios POST request to /api/tasks
   - Request includes:
     * JWT token in Authorization header
     * Task data in request body

3. NETWORK:
   - HTTP POST request sent to http://localhost:5001/api/tasks
   - Headers: Authorization: Bearer <token>
   - Body: { title, description, project, assignedTo, priority, dueDate, tags }

4. BACKEND (Express):
   - Request received by Express server
   - Passes through authMiddleware (protect)
   - Token verified, user authenticated
   - Routes to taskController.createTask()

5. CONTROLLER:
   - Validates project exists
   - Checks user has access to project
   - Creates new Task document
   - Saves to MongoDB

6. DATABASE (MongoDB):
   - Task document inserted into tasks collection
   - Returns inserted document with _id

7. RESPONSE:
   - Controller populates task with user/project data
   - Returns JSON response with status 201
   - Response: { _id, title, description, ... }

8. FRONTEND RECEIVES:
   - Axios promise resolves
   - Task added to local state
   - UI updates automatically
   - Modal closes
   - Success feedback shown


================================================================================
                    AUTHENTICATION FLOW
================================================================================

LOGIN PROCESS:
--------------
1. User enters email and password
2. Frontend sends POST to /api/auth/login
3. Backend finds user by email
4. Compares password using bcrypt
5. Generates JWT token (30-day expiry)
6. Returns user data + token
7. Frontend stores token in localStorage
8. Frontend stores user in Context
9. User redirected to dashboard
10. All subsequent requests include token

PROTECTED ROUTE ACCESS:
-----------------------
1. User tries to access /dashboard
2. ProtectedRoute component checks isAuthenticated
3. If not authenticated → redirect to /login
4. If authenticated → render Dashboard component
5. Dashboard makes API calls with token
6. Backend verifies token on each request
7. If token valid → return data
8. If token invalid → return 401 error
9. Frontend intercepts 401 → logout user
